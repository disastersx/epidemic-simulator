1. Descripcion del Problema

Contexto

Despues de vivir la pandemia del COVID-19, entendimos la importancia de modelar como se propagan las enfermedades infecciosas en una poblacion. Este proyecto implementa un simulador que permite visualizar y analizar la propagacion de una epidemia usando el modelo epidemiologico SIR (Susceptible-Infectado-Recuperado).

Motivacion

Las pandemias han demostrado ser uno de los mayores desafios de salud publica. Comprender como se propaga una enfermedad permite tomar decisiones informadas sobre medidas de contencion, vacunacion y distanciamiento social. Este simulador permite experimentar con diferentes parametros y observar sus efectos en tiempo real.

Objetivos

  • Modelar una red de contactos como un grafo no dirigido
  • Implementar el modelo SIR para simular estados de salud
  • Visualizar la propagacion usando algoritmo BFS
  • Generar reportes y graficos de la evolucion de la epidemia

2. Modelado del Problema

El Grafo de Contactos

Modelamos la poblacion como un grafo no dirigido G = (V, E) donde:

Nodos (V)

Cada nodo representa una persona de la poblacion. Tiene un ID unico, nombre y un estado de salud.

Aristas (E)

Cada arista representa un contacto fisico entre dos personas. Son no dirigidas (si A contacta a B, B contacta a A).

Sin Pesos

Las aristas no tienen peso. Todos los contactos tienen la misma probabilidad de contagio.

Estados del Modelo SIR

Estado Color Descripcion Transiciones
Susceptible (S) Azul Persona sana que puede infectarse S → I (con probabilidad p al contacto con infectado)
Infectado (I) Rojo Persona enferma que puede contagiar I → R (despues de d dias)
Recuperado (R) Verde Persona recuperada e inmune Estado final (no hay transiciones)

Diagrama del Modelo

Tasa de contagio (p) | v +-------------+ contacto +-------------+ d dias +-------------+ | SUSCEPTIBLE | ------------> | INFECTADO | -----------> | RECUPERADO | | (Azul) | | (Rojo) | | (Verde) | +-------------+ +-------------+ +-------------+ | | | | Puede infectarse | Puede contagiar | Inmune | si tiene contacto | a sus vecinos | (estado final) | con infectado | susceptibles | +-----------------------------+----------------------------+

3. Estructuras de Datos Utilizadas

3.1 Lista de Adyacencia (Estructura Principal del Grafo)

Implementamos el grafo usando una lista de adyacencia con la estructura Map<string, Set<string>> de JavaScript.

Justificacion

  • Eficiencia en espacio: O(V + E) vs O(V^2) de matriz de adyacencia
  • Redes dispersas: En redes sociales reales, cada persona tiene pocos contactos comparado con el total
  • Acceso rapido: O(1) para verificar si existe una arista usando Set
  • Iteracion eficiente: Recorrer vecinos de un nodo es O(grado)
// Implementacion de la lista de adyacencia
private adjacencyList: Map<string, Set<string>> = new Map();

// Agregar arista (no dirigida)
addEdge(source: string, target: string): void {
  if (!this.adjacencyList.has(source)) {
    this.adjacencyList.set(source, new Set());
  }
  if (!this.adjacencyList.has(target)) {
    this.adjacencyList.set(target, new Set());
  }
  this.adjacencyList.get(source)!.add(target);
  this.adjacencyList.get(target)!.add(source);
}

// Obtener vecinos
getNeighbors(nodeId: string): string[] {
  return Array.from(this.adjacencyList.get(nodeId) || []);
}

3.2 Arreglo de Nodos (nodeMap)

Usamos un Map<string, NodeData> para almacenar la informacion de cada nodo: 

interface NodeData {
  id: string;
  name: string;
  state: 'susceptible' | 'infected' | 'recovered';
  infectedDay: number;    // Dia en que se infecto
  infectedBy: string | null;  // Quien lo infecto (trazabilidad)
}

private nodeMap: Map<string, NodeData> = new Map();

Justificacion

  • Acceso O(1) a cualquier nodo por su ID
  • Almacena metadata necesaria para la simulacion (estado, dia de infeccion)
  • Permite trazabilidad de la cadena de contagios

3.3 Cola de Infectados (BFS)

Para la simulacion diaria usamos un arreglo como cola que contiene los nodos infectados activos: 

// Cola de infectados para BFS
const infectedQueue: string[] = this.getInfectedNodes().map(n => n.id);

// Procesamiento BFS por niveles (un dia = un nivel)
while (infectedQueue.length > 0) {
  const currentId = infectedQueue.shift()!;
  const neighbors = this.getNeighbors(currentId);
  
  for (const neighborId of neighbors) {
    const neighbor = this.nodeMap.get(neighborId)!;
    if (neighbor.state === 'susceptible') {
      if (Math.random() < this.infectionRate) {
        // Infectar vecino
        neighbor.state = 'infected';
        neighbor.infectedDay = this.currentDay;
      }
    }
  }
}

3.4 Arreglo de Historial (Reportes)

Almacenamos el estado de cada dia en un arreglo para generar graficos: 

interface DailyStats {
  day: number;
  susceptible: number;
  infected: number;
  recovered: number;
  newInfections: number;
  newRecoveries: number;
}

private history: DailyStats[] = [];

Comparacion de Estructuras

4. Operaciones Implementadas

4.1 Cargar Red desde Archivo

Lee un archivo JSON con nodos y aristas, construyendo el grafo.

loadFromJSON(data: NetworkFileFormat): void {
  // Limpiar grafo anterior
  this.adjacencyList.clear();
  this.nodeMap.clear();
  
  // Agregar nodos
  for (const node of data.nodes) {
    this.addNode(node.id, node.name);
  }
  
  // Agregar aristas
  for (const edge of data.edges) {
    this.addEdge(edge.source, edge.target);
  }
}

Complejidad: O(V + E)

4.2 Definir Infectados Iniciales

Marca uno o varios nodos como infectados para iniciar la simulacion.

setInitialInfected(nodeIds: string[]): void {
  for (const id of nodeIds) {
    const node = this.nodeMap.get(id);
    if (node && node.state === 'susceptible') {
      node.state = 'infected';
      node.infectedDay = 0;
      node.infectedBy = null; // Paciente cero
    }
  }
}

Complejidad: O(k) donde k = cantidad de infectados iniciales

4.3 Configurar Parametros

Establece la tasa de contagio y dias de recuperacion.

setParameters(infectionRate: number, recoveryDays: number): void {
  this.infectionRate = infectionRate;  // 0.0 a 1.0
  this.recoveryDays = recoveryDays;    // entero positivo
}

Complejidad: O(1)

4.4 Simular Un Dia (BFS de un nivel)

Avanza la simulacion un dia usando BFS desde los nodos infectados.

simulateOneDay(): SimulationResult {
  this.currentDay++;
  let newInfections = 0;
  let newRecoveries = 0;
  
  const infected = this.getInfectedNodes();
  
  // Fase 1: Intentar contagiar vecinos
  for (const node of infected) {
    const neighbors = this.getNeighbors(node.id);
    for (const neighborId of neighbors) {
      const neighbor = this.nodeMap.get(neighborId)!;
      if (neighbor.state === 'susceptible') {
        if (Math.random() < this.infectionRate) {
          neighbor.state = 'infected';
          neighbor.infectedDay = this.currentDay;
          neighbor.infectedBy = node.id;
          newInfections++;
        }
      }
    }
  }
  
  // Fase 2: Recuperar infectados antiguos
  for (const node of infected) {
    if (this.currentDay - node.infectedDay >= this.recoveryDays) {
      node.state = 'recovered';
      newRecoveries++;
    }
  }
  
  return { day: this.currentDay, newInfections, newRecoveries };
}

Complejidad: O(I * d) donde I = infectados, d = grado promedio

4.5 Simular Hasta Estabilizacion

Ejecuta la simulacion hasta que no queden infectados.

simulateUntilStable(): SimulationHistory {
  while (this.getInfectedCount() > 0) {
    this.simulateOneDay();
    this.recordDailyStats();
  }
  return this.history;
}

Complejidad: O(dias * I * d)

4.6 Generar Reporte Diario

Calcula y almacena estadisticas del dia actual.

getDailyReport(): DailyStats {
  return {
    day: this.currentDay,
    susceptible: this.getSusceptibleCount(),
    infected: this.getInfectedCount(),
    recovered: this.getRecoveredCount(),
    newInfections: this.todayNewInfections,
    newRecoveries: this.todayNewRecoveries,
    r0Estimate: this.calculateR0()
  };
}

Complejidad: O(V)

5. Casos de Prueba

Caso 1: Red Pequena (5 nodos lineales)

Entrada: Nodos: A - B - C - D - E (linea) Infectado inicial: A Tasa contagio: 100% Dias recuperacion: 3 Salida esperada: Dia 0: I=1, S=4, R=0 Dia 1: I=2, S=3, R=0 (B infectado) Dia 2: I=3, S=2, R=0 (C infectado) Dia 3: I=3, S=1, R=1 (D infectado, A recuperado) Dia 4: I=3, S=0, R=2 (E infectado, B recuperado) ... Final: I=0, S=0, R=5 (todos recuperados)

Caso 2: Red con Comunidades

Entrada: Grupo A: 1-2-3 (triangulo) Grupo B: 4-5-6 (triangulo) Puente: 3-4 Infectado inicial: 1 Tasa contagio: 50% Dias recuperacion: 5 Comportamiento esperado: - Primero se infecta el Grupo A - Luego salta al Grupo B por el nodo 3-4 - El contagio es aleatorio (50% probabilidad)

Caso 3: Red Grande (40 nodos, 5 comunidades)

Entrada: Red de Medellin con 40 personas en 5 grupos: - Familia Garcia (8 personas) - Empresa TechCo (8 personas) - Universidad (8 personas) - Barrio Laureles (8 personas) - Gimnasio (8 personas) 12 puentes inter-grupo Infectado inicial: Juan Garcia Tasa: 40%, Recuperacion: 5 dias Metricas esperadas: - Pico de infectados: dia 4-7 - Total infectados final: 60-90% de la poblacion - R0 estimado: 1.5 - 2.5

8. Limitaciones y Posibles Mejoras

Limitaciones Actuales

  • Sin periodo de incubacion: Los infectados contagian desde el dia 1
  • Tasa uniforme: Todos los contactos tienen la misma probabilidad de contagio
  • Sin mortalidad: El modelo no incluye muertes
  • Red estatica: Los contactos no cambian durante la simulacion
  • Sin vacunacion: No hay forma de inmunizar nodos

Posibles Mejoras

  • Modelo SEIR: Agregar estado "Expuesto" con periodo de incubacion
  • Aristas ponderadas: Diferentes probabilidades segun tipo de contacto
  • Vacunacion: Inmunizar nodos estrategicos
  • Cuarentena: Aislar nodos detectados como infectados
  • Redes dinamicas: Cambiar contactos segun el dia de la semana
  • Importar redes reales: Integrar con datos de redes sociales
  • Multiples enfermedades: Simular varias epidemias simultaneamente

Preguntas Frecuentes

Pregunta Respuesta
Por que lista de adyacencia y no matriz? Porque la red es dispersa. Usamos O(V+E) en vez de O(V^2) de memoria.
Cual es la complejidad del BFS? O(V+E) en general, pero nosotros solo recorremos desde infectados: O(I*d).
Que pasa si la tasa es 100%? Todos los susceptibles conectados a infectados se contagian cada dia.
Como calculan el R0? Promedio de nuevos infectados dividido por los infectados que los causaron.
Por que usaron TypeScript? Para tener tipado fuerte y evitar errores en tiempo de ejecucion.
